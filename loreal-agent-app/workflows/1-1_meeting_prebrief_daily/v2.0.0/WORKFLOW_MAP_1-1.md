## Workflow 地图 - meeting_prebrief_daily v2.0.0（1-1）

本文档按“输入 → 步骤 → 输出 → 缺口”的结构，明确每个步骤所需数据与完成度。

### 说明
- 状态：`已实现`（available）、`部分实现`（partial）、`缺失`（missing）
- 数据来源：`输入`（调用方提供）、`mysql`（装配）、`规则`（派生）、`llm`（可选）
- `部分实现`/“best-effort”的含义：会尽力从现有表聚合，但**口径不完整、字段可能缺失、缺失时会返回 0/空数组，不会报错**。

### 输入

**最小输入（默认 mysql 装配）**
- `store_id`
- `biz_date`

**可选输入**
- `store_name`, `data_cutoff_time`
- `mtd.*_target`（月度目标）
- 其它结构化字段：`staff_stats`, `customer_summary`, `key_items_mtd`, `task_execution`
- 外部目标与计划输入（XLS）：`daily_targets`, `staff_targets`, `schedule_plan`（来源 `reference/*.xlsx`，后续可替换为接口/系统接入）

### 步骤地图

| 步骤 | 描述 | 数据来源 | 状态 | 备注 |
| --- | --- | --- | --- | --- |
| S0 请求接收 | 接收原始 input + context | 输入 | 已实现 | 入口。 |
| S1 规范化 | MySQL 装配 + input 覆盖合并，剥离 context | mysql + 输入 | 已实现 | 默认装配。 |
| S2 完整性检查 | 校验必填字段 | 输入或 mysql | 已实现 | `his` 由装配补齐，调用方可不传。 |
| S3A 缺失信息处理 | 生成澄清请求（可选） | 规则 | 缺失 | 暂无显式输出。 |
| S4 执行 | 计算 facts/risks/checklist | mysql + 输入 + 规则 | 已实现 | 多字段为 best-effort。 |
| S4.5 智能总结 | 基于 facts/risks/checklist 生成总结要点 | 规则 + llm | 已实现 | LLM 失败时回退规则总结。 |
| S5 结果校验 | 输出 schema 校验 | 规则 | 已实现 | 不通过直接失败。 |
| S6 交付与落盘 | 渲染报告 + 持久化 | 规则 | 已实现 | 写入 `reports/`。 |

### 输出

**结构化输出（JSON）**
- `facts_recap`, `tomorrow_list`, `risks`, `checklist`, `data_quality`
- 可选：`agent_summary`
- 可选：`agent_risk_summary`, `agent_staff_summary`, `agent_customer_summary`, `agent_key_items_summary`

**人类可读输出**
- `report_md`（Markdown 报告）
- 落盘文件：`reports/briefing_YYYYMMDD.md`

### 输出（按模板章节）

每个指标的计算口径与字段映射统一放在 `loreal-agent-app/workflows/1-1_meeting_prebrief_daily/v2.0.0/METRICS_MAP_1-1.md`。

#### 今日经营摘要
- 今日开单、今日消耗
- 今日预约人数、到店人数、成交人数
- 月度累计开单、月度累计消耗、月度时间进度
- 月度开单指标完成度、月度消耗指标完成度

#### 智能总结
- 总结要点

#### 各健康管理人完成情况
- 管理人姓名
- 今日开单/消耗、本月累计开单/消耗
- 达成率/目标、R12 回购率
- 智能总结

#### 顾客摘要
- 新客人数/GMV、新客来源分层
- 老带新核验/美丽基金
- 老客人数/GMV
- 单项目顾客、VIP/VVIP
- 智能总结

#### 关键品项完成（本月至今）
- 品项开单/消耗金额、品项人数
- WOW/同期对比、单次比例/复购提示
- 扫码购明细
- 智能总结

#### 任务执行情况
- 回访完成率、对比照发送完成率
- 术后提醒完成率
- AI 面诊记录生成率、病历完成比例、处方开具比例
- 名单级明细
- 群内交接比例/对话质量

#### 明日生意准备
- 明日预约人数与清单
- 明日业绩目标
- 预约分群
- 当班医生/护士与人手风险

#### 接下来几天
- 专家日目标/预约
- 未来 7 天目标与预约量
- 客单差距测算
- 单次客回店邀约
- VIP 维护到店
